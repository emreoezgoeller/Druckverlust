# Sprint 20.04 – Roadmap, Feedback und Changelog

## Ziel

Die Weiterentwicklung von Druckverlust Pro sichtbar machen, ohne die Produktseite zu überladen. Rückmeldungen sollen strukturiert vorbereitet werden können, ohne bereits einen Server oder Login zu benötigen.

## Umsetzung

- Kompakter Roadmap-Bereich auf `index.html` und `produkt.html`.
- Direkte Links zur Roadmap und Feedback-Vorlage in der Tool-Hilfe.
- Tool-Hilfe um drei Bereiche erweitert: Roadmap, Feedback, Versionshistorie.
- Feedback-Vorlage kann in die Zwischenablage kopiert werden.
- Zusätzliche CSS-Datei `src/ui/phase20_04.css`, damit bestehende Shell-Styles nicht unnötig verändert werden.
- Cache-Busting und zentrale App-Version auf 20.04 erhöht.

## Hinweis

Es werden noch keine Rückmeldungen an einen Server gesendet. Die Funktion erstellt nur eine strukturierte Vorlage, die anschliessend per E-Mail oder anderem Kanal verschickt werden kann.
